# Pipe Gallery Node 项目结构框架

## 项目概述
这是一个基于 **RT-Thread** 实时操作系统的 **STM32H7** 嵌入式项目，主要用于管道廊道监测节点系统。项目集成了多种传感器数据采集、Modbus通信、WiFi/MQTT物联网通信等功能。

## 二、核心功能模块

### 2.1 应用程序层 (applications/)

#### **main.c** - 主程序入口
- 初始化检查机制（防止重复初始化）
- 创建并启动多个线程：
  - `uart5_thread`: UART5串口通信线程
  - `get_methane_thread`: 甲烷传感器数据采集线程
  - （注释掉的线程：Modbus主站轮询、UART4、WiFi等）

#### **NodeApp** - 节点数据结构
```c
typedef struct Node {
    float CH1_A[3];      // 三相电流（AC、BC、CA）
    float Flow;           // 流量
    char Flame;           // 火焰传感器状态（1有火/0无火）
    float Methane;        // 甲烷浓度
} Node;
```
- 支持多个节点（`NODENUM = 2`）

#### **uartApp** - 串口通信模块
- **UART4**: 接收缓冲区256字节
- **UART5**: 接收缓冲区128字节
- 功能：
  - 串口初始化
  - 数据发送/接收
  - 接收回调处理
  - 缓冲区管理

#### **MethaneSensorApp** - 甲烷传感器模块
- 实时采集甲烷浓度数据
- 独立线程运行：`get_methane_thread_entry()`

#### **fireSensorApp** - 火焰传感器模块
- 检测火焰状态（BOOL类型）

#### **freeModbusApp** - Modbus通信模块
- **主站功能**：
  - 支持多个从站设备
  - 轮询周期：500ms
  - 读取/写入保持寄存器和输入寄存器
- **从站设备**：
  - **电表（AMMETER）**: 从站地址2，读取三相电流（寄存器0x10E开始，6个寄存器）
  - **水表（WATERMETER）**: 从站地址1，读取流量（寄存器1开始，2个寄存器）
- **通信参数**：
  - 波特率：9600
  - 校验：偶校验（MB_PAR_EVEN）
  - 端口：PORT_NUM = 2

#### **wifiApp** - WiFi/MQTT物联网模块
- **WiFi配置**：
  - SSID: "Win10-2018PYBHT 2565"
  - Password: "1&H09h78"
- **MQTT配置**：
  - 服务器：华为云IoT平台
  - 地址：`48931f72a4.st1.iotda-device.cn-north-4.myhuaweicloud.com`
  - 端口：1883
  - 客户端ID：`67bedcb724d772325520a4c7_pipe_1_0_0_2025022804`
- **功能**：
  - WiFi连接管理
  - MQTT连接与订阅
  - 数据上报（节点属性：流量、火焰、甲烷、电流等）

## 四、通信协议

### 4.1 Modbus RTU
- **协议栈**: FreeModbus
- **模式**: 主站模式
- **从站设备**:
  1. 电表（地址2）- 读取三相电流
  2. 水表（地址1）- 读取流量
- **通信参数**: 9600-8-E-1

### 4.2 UART串口
- **UART4**: 256字节接收缓冲区
- **UART5**: 128字节接收缓冲区
- 支持中断接收和线程处理

### 4.3 MQTT物联网
- **平台**: 华为云IoT平台
- **协议**: MQTT 3.1.1
- **数据格式**: JSON
- **主题**: 
  - 上报：`$oc/devices/{device_id}/sys/properties/report`
  - 订阅：`oc/devices/{device_id}/sys/properties/report`

## 五、数据流

### 5.1 传感器数据采集流程

```
传感器硬件
    ↓
驱动层 (drv_*.c)
    ↓
应用层线程
    ↓
Node数据结构
    ↓
数据处理/上报
```

### 5.2 Modbus数据采集流程

```
Modbus从站设备（电表/水表）
    ↓
UART串口（RTU模式）
    ↓
FreeModbus协议栈
    ↓
freeModbusApp应用层
    ↓
Node数据结构更新
```

### 5.3 数据上报流程

```
Node数据结构
    ↓
wifiApp模块
    ↓
JSON格式化
    ↓
MQTT发布
    ↓
华为云IoT平台
```

## 十、总结

这是一个**管道廊道监测节点系统**，主要功能包括：
1. **多传感器数据采集**（甲烷、火焰、电流、流量）
2. **Modbus工业通信**（与电表、水表通信）
3. **串口通信**（UART4/UART5）
4. **物联网数据上报**（WiFi/MQTT，当前已注释）

项目采用**分层架构设计**，从硬件驱动到应用层逻辑清晰，便于维护和扩展。基于RT-Thread实时操作系统，保证了系统的实时性和可靠性。
